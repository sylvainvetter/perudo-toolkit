# M2 — Action Recommender

## Overview

M2 recommends the best action (`raise`, `liar`, or `exact`) for the current player
given the full game state. It uses M1 for probability estimates.

## Decision logic (v1)

```
p_true  = P(current bid is satisfied | own dice)
p_exact = P(current bid is exact     | own dice)

if p_true  < threshold_liar                       → Liar
if p_exact > threshold_exact AND exact available  → Exact
otherwise                                         → best legal Raise
```

Default thresholds (calibrated by M3 in S7):
- `threshold_liar  = 0.25`
- `threshold_exact = 0.35`

Raise score: `p_true(raised_bid) − lambda_risk × max(0, q − E[T])`

## API

```python
from perudo.core.types import Bid, GameState, Player, RoundState
from perudo.m2 import recommend, RecommenderConfig

rec = recommend(game_state)
print(rec.best_action)          # "raise" | "liar" | "exact"
print(rec.bid_if_raise)         # Bid or None
print(rec.rationale.p_current_bid_true)   # P(bid true)
print(rec.rationale.expected_total)       # E[T]
```

### Custom thresholds

```python
cfg = RecommenderConfig(threshold_liar=0.30, threshold_exact=0.40, lambda_risk=0.5)
rec = recommend(game_state, config=cfg)
```

## Output structure

| Field | Type | Meaning |
|---|---|---|
| `best_action` | `"raise"/"liar"/"exact"` | Recommended action |
| `bid_if_raise` | `Bid \| None` | The raise to make (None if liar/exact) |
| `rationale.p_current_bid_true` | float | P(current bid satisfied) |
| `rationale.p_current_bid_exact` | float | P(current bid exact) |
| `rationale.expected_total` | float | E[T] given own dice |
| `rationale.alternatives_considered` | list[RaiseCandidate] | All evaluated raises |
| `rationale.score` | float | Decision confidence score |

## Notes

- Opening bid (no previous bid): always returns `"raise"`, rationale fields are 0
- Opponent dice **values** are unknown; only their **count** affects the estimates
- `exact` is only recommended if `player.exact_used == False`
